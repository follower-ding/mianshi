# 简历模块用户帮助

> 面向求职者的简明说明。开发/部署文档见 [RESUME-LAUNCH-PLAN.md](./RESUME-LAUNCH-PLAN.md)、[POSTGRESQL_GUIDE.md](./POSTGRESQL_GUIDE.md)。

## 导入说明

### 三步向导

1. **上传或粘贴原文** — 支持 PDF、DOCX、TXT、Markdown（单文件 ≤ 10MB）
2. **确认提取文本** — 核对提取结果，可编辑后再继续
3. **智能识别并对照** — 在弹窗中对比原文与识别结果，确认后应用

### 常见情况

| 情况 | 说明 |
|------|------|
| 不能 1:1 还原 PDF 版式 | 正常。系统会重新排版到模板中 |
| 扫描版 PDF 失败 | 无法 OCR 时请粘贴文本或使用可复制 PDF |
| 字段错位 | 在对照页修正，或导入后在排版编辑中改 |

应用内入口：**导入优化** → 页内「导入说明与演示模式」链接，或 `/resume/help#import`。

## 演示模式说明

未配置 `LLM_API_KEY` 时：

- 生成/识别/优化使用**规则模板**（非大模型）
- 界面显示「演示模式」徽章，关键操作前会确认
- 适合体验流程，**不建议**直接用于投递

配置方式：在 `mianshi-api/.env` 设置 `LLM_API_KEY` 后重启 API。

应用内入口：`/resume/help#demo`。

## 导出与分享

- **HR 投递**：导出 PDF（浏览器或服务端）或创建**公开只读链接**（`/r/:token`）
- **本人编辑**：复制编辑页链接（需登录）
- **图片**：PNG/JPG 适合预览；正式投递优先 PDF

## 获取帮助

- 应用内：`/resume/help`
- API 健康：`GET /api/resumes/health`（是否 demo、是否 PG 同步）
